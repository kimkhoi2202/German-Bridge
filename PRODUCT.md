# Product

## Register

product

## Users

German Bridge players who want a focused, polished digital table for playing one-hand matches against bots. They already know the game or are willing to learn outside the main play flow, so the interface should prioritize seeing the table, hand, bids, tricks, and history without instructional clutter.

## Product Purpose

German Bridge is a premium-feeling card game app for setting up a match, bidding, playing tricks, and reviewing scores. Success means the player can read the table state at a glance, make decisions quickly, and trust that the layout behaves like a real game client on desktop and mobile.

## Brand Personality

Confident, refined, quietly competitive. The app should feel like an official high-quality game with serious craft, not a demo, mockup, casino gimmick, or generic AI-generated dashboard.

## Anti-references

Avoid neon borders, over-glowing cards, loud yellow/red treatments, heavy drop shadows, visible how-to-play copy during gameplay, default browser controls, crowded overlapping cards, and UI that looks like a generated concept image instead of a playable product.

## Design Principles

1. The table is the product: keep game state, players, hand, bidding, and played-card order visible whenever possible.
2. Game screens should feel locked and stable, but never at the cost of hiding required controls.
3. Use calm luxury: quiet emerald and graphite surfaces, restrained accents, softened borders, and purposeful depth.
4. Familiar controls beat invented ones: steppers, segmented controls, toggles, rails, and nav items should act predictably.
5. Motion should communicate state smoothly and quickly, never call attention to itself or animate layout in a janky way.

## Accessibility & Inclusion

Target WCAG AA where it does not conflict with game-specific interaction needs. Avoid default focus rings when they clash visually, but replace them with intentional custom focus states. Maintain reduced-motion support, adequate touch targets, readable contrast, and semantic labels for controls and cards.
